# client/src/i18n.js

## What is this file?

Initialises **i18next** — the internationalisation (translation) library used to display text in English, French, or Spanish.

**i18n** is short for "internationalisation" (18 letters between the 'i' and the 'n').

## What this file does

```js
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';
import en from './locales/en/translation.json';
import fr from './locales/fr/translation.json';
import es from './locales/es/translation.json';

i18n.use(initReactI18next).init({
    resources: {
        en: { translation: en },
        fr: { translation: fr },
        es: { translation: es },
    },
    lng: localStorage.getItem('makeventory_lang') || 'en',
    fallbackLng: 'en',
    interpolation: { escapeValue: false },
});
```

## Configuration explained

### `resources`

Loads all three translation files into i18next's memory at startup. This means translations are available immediately without additional network requests.

### `lng`

The active language. On startup, it reads from `localStorage` under the key `makeventory_lang`. This persists the user's language preference across browser sessions.

If no preference is stored, it defaults to `'en'` (English).

### `fallbackLng: 'en'`

If a translation key is missing in the active language, fall back to English rather than showing the raw key string.

### `interpolation: { escapeValue: false }`

React already escapes HTML in text, so i18next doesn't need to escape it again. Disabling this prevents double-escaping.

## How translations are used in components

Any component can access translations using the `useTranslation` hook:

```jsx
import { useTranslation } from 'react-i18next';

function MyComponent() {
    const { t } = useTranslation();
    return <h1>{t('dashboard.title')}</h1>;
}
```

`t('dashboard.title')` looks up the key `dashboard.title` in the current language's translation file.

## Changing language at runtime

When the user changes the language in Language & Region settings:
1. `LanguageRegionPage.jsx` calls the API to update `GlobalSettings`
2. `GlobalSettingsContext.jsx` calls `i18n.changeLanguage(newLanguage)`
3. All components re-render with the new language immediately

## Relationship to other files

- Translation JSON files: `locales/en/translation.json`, `locales/fr/translation.json`, `locales/es/translation.json`
- Language is persisted and synced by `GlobalSettingsContext.jsx`
- Must be imported before `App.jsx` (done in `main.jsx`)
- Every UI component uses the `t()` function from this library
